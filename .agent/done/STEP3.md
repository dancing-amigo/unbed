# STEP3: ローカル保存と設定管理を実装する

## 目的
- アラーム設定と進行中セッションを端末内で一貫して管理できるようにする。
- アプリ再起動や端末再起動後の復元に備える。

## 実装内容
- Room の Entity, DAO, Database を定義する。
- `AlarmConfig` 保存テーブルを作る。
- `AlarmSession` 保存テーブルを作る。
- 現在の進行中セッションを一意に扱う repository を実装する。
- `AlarmSession` に再鳴動回数と終了理由を保存できるようにする。
- 次回発火時刻計算ロジックを use case として実装し、通常アラーム予約と進行中セッションの再発火を区別して扱う。
- 曜日繰り返しの表現を保存可能な形式に落とす。

## 完了条件
- 単一アラームの設定保存と読み出しができる。
- 発火中またはスヌーズ中セッションを復元できる。
- 次回発火時刻をローカルデータから計算できる。
- 進行中セッションの回数上限判定と終了理由が保存状態から復元できる。

## 注意点
- MVP は単一アラーム前提だが、スキーマは複数対応へ拡張可能にしておく。
- `AlarmConfig` と `AlarmSession` を混同しない。
