# STEP5: Androidの発火基盤を実装する

## 目的
- アプリを閉じていても指定時刻にアラーム発火処理を開始できるようにする。

## 実装内容
- `AlarmManager` を使った次回発火予約を実装する。
- exact alarm 権限の有無を判定し、必要時は設定画面導線を用意する。
- `BroadcastReceiver` で予約イベントを受信する。
- Receiver から発火セッション生成と通知表示処理へ接続する。
- アラーム再予約ロジックを共通化する。
- 進行中セッションの再発火と通常アラーム時刻が競合した場合は、通常アラームを優先して新しいセッションへ切り替える。

## 完了条件
- 指定した時刻に receiver が起動する。
- アプリが foreground でなくても発火処理へ進める。
- 予約の上書きとキャンセルができる。
- 進行中セッションと次回通常予約の競合時に、優先ルールどおり新しいセッションへ切り替わる。

## 注意点
- Android バージョン差分とメーカー制御で完全保証はできないため、案内導線も同時に設計する。
