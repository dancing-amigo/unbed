# STEP12: MVP仕上げ

## 目的
- 実際に試せる MVP としての完成度を上げる。

## 実装内容
- 文言、エラー表示、権限不足時のガイドを整える。
- ログとクラッシュ収集を入れる。
- 実機で発火、QR、スヌーズ、再鳴動を確認する。
- 端末差分の注意事項を整理する。
- MVP の制約を README と設計書に反映する。
- 固定 QR、カメラ必須、解除不能時の救済導線なし、再鳴動上限 10 回を既知制約として明記する。

## 完了条件
- 主要フローを手元実機で通せる。
- ユーザーが迷いやすい箇所に説明がある。
- 既知制約が明文化されている。

## 注意点
- 「手動解除で抜けられる」点は MVP 制約としてユーザーにも開発者にも共有する。
- カメラ故障や端末異常で QR が使えない場合の未救済制約を隠さない。
