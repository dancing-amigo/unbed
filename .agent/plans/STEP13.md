# STEP13: 将来拡張の準備を残す

## 目的
- MVP 完成後に歩数解除、固有 QR、iOS 追加へ進みやすい構造を残す。

## 実装内容
- `ReleaseCondition` を interface または sealed hierarchy として整理する。
- `manual_release` 実装を差し替え可能な形にする。
- `step_count_release` 導入時に必要なデータ項目を洗い出す。
- `packages/domain-spec` に将来仕様を追記する。
- ユーザー固有 QR や API 導入用の契約境界を文書化する。
- iOS 実装時に再利用するドメイン仕様を整理する。

## 完了条件
- MVP コードが将来機能の足かせになりにくい。
- 解除条件追加時に UI と OS 実装の変更点が限定される。
- 次フェーズの設計入力が文書として残っている。

## 注意点
- 未来のために抽象化しすぎず、MVP で必要な範囲に留める。
